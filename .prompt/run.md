## agentmap

Create a file named `agentmap.md` in the project root directory. This file serves as a guide for the agent when executing tasks, eliminating the need for complex queries each time. The file contains:

- The overall architecture of the project
- Summaries of each functional module
- The function and description of all files

A file tree is generated by scanning all files in the project, with descriptions added for each folder and file. This is represented as a JSON object within a code block for better structure and readability:

- The function implemented by the folder/file
- The organizational structure used
- Folder/file naming conventions
- For each file: its input (external dependencies), output (what it provides externally), role (its role in the overall system), and a brief description of the implemented function.

The file tree is updated whenever the content of a folder or file changes.
