## V3

引入 PFC（LMDB）和 STR（LanceDB）后，你的数据流将从“线性搜索”转变为“双路/多路反馈控制”架构。

PFC 作为“指挥中心”利用 LMDB 的极速读取（内存映射 I/O）来拦截任务，STR 负责基于奖励的习惯反应，而你原有的海马体逻辑则沉淀为“深思熟虑”的后方支持。

Polywise 脑机架构数据流：

```bash
          用户输入 (Input)
                ↓
┌──────────────────────────────────────────────┐
│  Phase 0: PFC 任务门控 (Executive Gating)    │
│  - LMDB 读取: 当前 Session Context & 活跃目标  │
│  - 状态更新: 将新输入压入工作记忆栈          │
└──────────────────────────────────────────────┘
      ↓                      ↘
[ 命中习惯? ]             [ 未命中 / 复杂任务 ]
      ↓                      ↘
┌─────────────────────────┐    ┌──────────────────────────────────────────┐

      ↓                        ┌──────────────────────────────────────────┐

      ↓                        │ - 知识库补全                             │
      ↓                        └──────────────────────────────────────────┘
      ↓                                      ↓
┌──────────────────────────────────────────────────────────────────────────┐
│ Phase 3: 聚合与权重平衡 (Aggregate & Weighting)                          │
│ - 冲突检测: 如果 STR 结果与 HIP 冲突，PFC 根据置信度仲裁                 │

                │ - 计算奖励 (Reward)              │
```

技术栈实施细节：

1. **PFC (LMDB - 内存映射存储)**:
      - **存储内容**：`current_task_id`, `working_memory_tokens`, `active_goals`。
      - **优势**：LMDB 在处理这类小体积、高频读写的数据时，比任何数据库都快，且天然支持 ACID 保证上下文不丢失。

2. **STR (LanceDB - 策略表)**:
      - **存储内容**：`embedding(state) | action_type | reward_score | timestamp`。
      - **逻辑**：它像是一个“快捷方式库”。如果 `state` 相似度高且 `reward_score` 表现优异，直接从这里取结果，绕过复杂的图遍历（Phase 1-2）。

3. **HIP (pglite - 联想图谱)**:
      - **作用**：它依然是你的核心。当机器人遇到“没见过”的指令，PFC 会下令让 HIP 进行大规模的联想和外部搜索。

核心改变：自学习反馈循环

引入该架构后，你的 **Phase 5** 是系统进化的关键。

- **信用分配**：如果用户说“做的好”，PFC 会从 LMDB 中取出 `last_task_id`，然后在 STR 中找到对应的 LanceDB 记录并增加其 `reward_score`。
- **热点维持**：通过这种反馈，你成功解决了“调用缓存不调用海马体”的问题——因为**每一次 STR 的成功调用都会被记录并在“睡眠期”反哺给海马体**。

## V2

添加 COT：基于多层结构的深度思考链

```bash
Depth 0: Initial query → result[0]
         ↓
Depth 1: Emerged query (based on result[0].title)
         Recall depth +1 → Search → Aggregate → Rerank → Stimulate
         ↓
Depth 2: Emerged query (based on new result[0].title)
         Recall depth +2 → Search → Aggregate → Rerank → Stimulate
         ↓
Depth 3: Continue until max depth...
```
