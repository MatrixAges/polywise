## Sophisticated Neural Network Memory Mechanism

- Global inhibition supersedes local competition: The biological neocortex/hippocampus relies heavily on local inhibitory loops, lateral inhibition, and winner-take-all mechanisms to boost resolution; pure global inhibition is more akin to "global blunting," which under high load leads to uniformly degraded retrieval or a bias where "only strong connections survive, while details are lost."
- Arousal modulation is overly monolithic: Estimating novelty based solely on "similarity to existing content" and linearly tuning learning (`packages/polywise/src/Polywise.ts:205`) misses key neuromodulatory mechanisms: reward prediction error (dopamine), task relevance (PFC gating), and the inverted-U (Yerkes–Dodson law: excessive arousal impairs encoding/retrieval). The likely result: failing to reinforce what should be strengthened, and inadvertently solidifying what should be suppressed.
- Replay "priorities" are not experience-dependent: Current sleep replay is too random (`packages/polywise/src/sql/Brain.ts:64`), whereas biological replay is strongly biased toward: recent experiences, reward/surprise, uncompleted goals, and fragile segments; random replay leads to meaningless consolidation, or even elevating noisy associations.
- Forgetting mechanisms lean "global/threshold-triggered" rather than "competition/interference-driven": Much of human forgetting stems from interference and retrieval competition (including retrieval-induced forgetting); relying mainly on global decay/pruning invites phase-change behavior where "either nothing is forgotten or everything is forgotten," lacking fine-grained, controllable forgetting.
- Absence of "truth maintenance/conflict monitoring" loops: Biological memory does not guarantee truth, but prefrontal monitoring, evidence integration, and source memory suppress confident errors. The current system resembles reinforcing retrieved content (`packages/polywise/src/utils/ranking.ts:102`), risking a self-fulfilling loop: the more something is retrieved, the more it is consolidated, even if it is a mismatch.
- Lack of "episodic temporal structure": The hippocampus is crucial for sequences and temporal context (time cells / sequence replay); the system can currently trace back to the article level but lacks dedicated representations for "event order/time windows/state," impacting the stable retrieval of plans, causal chains, and procedural knowledge.

- Introduce Complementary Learning Systems (CLS) hierarchy: Explicitly distinguish "fast-learning hippocampal episodic indices" from "slow-learning neocortical semantic graphs," allowing the fast system to store first while the slow system abstracts stable edges via offline consolidation; this prevents direct contamination of the semantic net by fast system updates.
- Implement episodic binding + pattern separation: Split representations of "the same concept across different contexts" (contextualized nodes/edges) and use context gating during recall to reduce crosstalk; this corresponds to biological pattern separation / context gating.
- Shift replay from random to "prioritized experience replay": Prioritize replay based on recency, prediction error/surprise, goal relevance, conflict level, and forgetting risk; mirroring known bias patterns in hippocampal replay.
- Add conflict detection and source memory: Attach evidence counts, source reliability, and mutual exclusivity to every item/edge; monitor before reinforcement during retrieval (corresponding to PFC monitoring and source memory).
- Introduce competitive forgetting: Apply retrieval competition and inhibition to similar memories (an engineering approximation of retrieval-induced forgetting), enabling the system to "selectively forget the incorrect/irrelevant" rather than relying on global decay.
- Complement evaluation with neural/cognitive phenomena: Validate mechanisms against benchmarks rather than just retrieval relevance, e.g., spacing effect, context-dependent memory, interference, DRM false memory, reward-biased replay; the existing `packages/polywise/test/datasets/*` is well-suited for expansion into regression tests covering these paradigms.
