长期记忆与日记，新建一个 memory的pglite schema:
长期记忆，用原始纯文本和向量存储的形式保存，有限容量，来源于用户的输入，通过polywise.getLongMemory()获取纯文本，它是自更新的，需要注意的是它的容量是有限的，所以要设计出一套机制，但达到文本容量上限之后，丢弃掉长期未“谈及”的关联记忆

日记，当自动进行SLEEPING，为离上一轮的SLEEPING之间的所有内容生成日记，日记以 YYYY-MM-DD HH:mm:ss作为可查询变量进行保存，保存也是原始纯文本和向量存储共存，可通过 polywise.getDailyMemory('YYYY-MM-DD HH:mm:ss') 进行查询，返回一个{prev,next}对象，代表查询时间点的上一个和下一个日记

在整个系统中，用于立刻反应形成rules的优先级最高，其次就是就是brain，然后是长期记忆，再然后是日记，最后是knowledge，在query时（上述数据源都要query）然后按照优先级进行rerank加权

需要注意的是，时间离现在越远，加权值越低（rules和长期记忆不受这条规则影响），长期记忆和日记都需要添加export interface FiltersArgs {
idol_id?: string
root_ids?: string[]
metrics_ids?: string[]
}作为数据筛选支持

长期记忆和日记解决的是，类脑神经网络过于专注当下的（活跃度都是当下优先），缺少长期规划能力和时间维度的深度回忆和“翻找日记”的能力

Generative Agents (Stanford, 2023):
论文提出了 Memory Stream（记忆流） 架构。它同样使用了“重要性、关联性、及时性”三个维度进行评分，这与你的长期记忆筛选机制不谋而合。
Complementary Learning Systems (CLS) Theory:
心理学经典理论。认为大脑有两个学习系统：海马体（负责快速学习新事物，类似你的日记/实时输入）和大脑皮层（负责整合长期知识，类似你的长期记忆）。
MemGPT (2023):
该研究探讨了如何通过“内存管理”让 LLM 拥有无限上下文。它证明了将记忆分为 Main Context 和 External Storage（并手动进行换入换出）是提高长期规划能力的最佳路径。
