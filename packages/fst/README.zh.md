# FST (Full Self Thinking)

一个旨在实现持续思考、上下文可控且具备自主执行能力的智能体框架。

## 核心理念

FST 认为智能体应当是真正自主且持久的。通过引入基于文件的信息交换机制和有限上下文状态机，FST 确保了会话在长时间运行中依然保持可控、准确且经济。

## 技术栈

- **[pi-mono](https://github.com/badlogic/pi-mono)**：负责底层系统交互与执行。
- **Vercel AI SDK**：对接多模型 Provider 并管理标准化的对话会话。
- **[mingo](https://github.com/kofrasa/mingo)**：通过模型的结构化输出驱动“智能上下文”管理。
- **Polywise**：集成强化学习（RL）记忆引擎，实现高保真信息检索。

## 核心特性

- **文件化持久化**：无需数据库。每个对话拥有独立的文件夹，所有会话均以文件形式持久化存储。
- **有限上下文状态机**：基于 Mingo 的结构化上下文管理，支持 Undo/Redo 并持久化至磁盘。
- **引用式加载**：通过引用往上下文中加载“关键信息”（自动阅读并摘要），确保上下文始终精简有效。

- **强化学习记忆**：深度集成 Polywise 记忆引擎，实现信息准确、非无限叠加的单次 Session 对话。
- **自主执行**：支持不间断的自驱动任务处理，避免上下文失控导致的性能衰减。

## 可用性支持

- **智能模型路由**：通过廉价模型（如 Gemini Flash）或本地模型进行前置路由，智能分流任务以优化性能与成本。
- **经济安全性**：支持为单个模型或全局设置成本上限，基于 Token 计算实时监控，超额自动停用以保护用户钱包。
- **高可用性**：内置失败回退机制，当某个模型不可用时自动切换，确保工作流不中断。

## 快速开始

```bash
pnpm install
```

## 架构说明

FST 将“智能上下文”视为一个状态机。它不再盲目堆叠 Token，而是通过按需加载和引用机制，在有限的窗口内维持最高质量的信息密度。
