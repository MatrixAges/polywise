# FST (Full Self Thinking)

一个具有结构化有限上下文管理的自执行、不间断自主智能体。

## 核心思想

FST 旨在成为一个真正的自主思考智能体。它通过引入持久化的、基于文件的思考环境和结构化的有限上下文状态机，打破了传统的无状态对话模式。

### 1. 有限上下文状态机

上下文不再是无限的流，而是一个受管理的状态。利用 [mingo](https://github.com/kofrasa/mingo)，FST 维护一个结构化的“智能上下文”，支持：

- **撤销/重做 (Undo/Redo)**：在思考状态之间导航。
- **选择性加载**：通过引用动态加载“关键信息”。
- **自动摘要**：自动对引用内容进行摘要，确保活跃上下文保持在限制范围内。

### 2. 基于文件的持久化

FST 直接在文件系统上操作，无需复杂的数据库：

- **对话文件夹**：每个对话都有其独立的专用目录。
- **文件即会话**：所有的会话数据和思考状态都持久化为结构化文件。
- **信息交换**：使用文件作为状态同步和信息共享的核心机制。

### 3. 持续执行

与反应式机器人不同，FST 专为不间断运行而设计。它利用 [Polywise](@packages/polywise) 强化学习记忆引擎来：

- 在长期运行的会话中保持高度的信息准确性。
- 通过智能记忆巩固防止上下文爆炸。
- 在无需用户持续干预的情况下执行自我驱动的任务。

## 架构

FST 整合了多个高性能模块来实现其目标：

- **系统交互**：使用 [pi-mono](https://github.com/badlogic/pi-mono) 进行底层系统访问和控制。
- **模型管理**：使用 [Vercel AI SDK](https://sdk.vercel.ai/) 管理多个 LLM 提供者和会话。
- **上下文引擎**：使用 [mingo](https://github.com/kofrasa/mingo) 进行结构化对象操作和状态管理。
- **记忆引擎**：使用 [Polywise](@packages/polywise) 进行基于知识图谱的记忆和强化学习驱动的检索。

## 技术栈

- **平台**：Node.js / TypeScript
- **上下文**：mingo
- **AI 核心**：Vercel AI SDK
- **系统**：pi-mono
- **记忆**：Polywise

## 应用场景

### 自主研究

可以给 FST 分配一个主题，它会创建一个思考文件夹，并通过读取文件、总结发现并更新其内部记忆来持续进行研究。

### 自我纠错开发

通过将思考状态持久化为状态机，FST 在逻辑陷入僵局时可以回溯并尝试替代路径，就像人类开发人员一样。
